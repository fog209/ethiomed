# WardReady — Architecture Review

## Methodology

## 1. Circular Dependencies

### Finding: WeaknessService Migration in Query Path
- **Severity:** MEDIUM
- **Files:** `lib/features/quiz/weakness_service.dart:12-36`, `lib/features/articles/presentation/article_detail_screen.dart:123`
- **Explanation:** `getWeakFields()` triggers `_ensureLastQualityColumn()` which modifies the database schema DURING a read query
- **Why it matters:** Read operations should not have side effects; schema migration during user-facing operation creates unpredictable performance
- **Fix:** Run schema migrations at app startup, not during UI reads
- **Effort:** 2h

## 2. Feature Coupling

### Finding: QuizNotifier Tri-Service Dependency
- **Severity:** HIGH
- **Files:** `lib/features/quiz/quiz_notifier.dart:16-18`
- **Explanation:** `QuizNotifier` holds references to `QuizRepository`, `QuizSyncService`, and `SpacedRepetitionService` — if one throws, all fail
- **Why it matters:** Breaking change in any service breaks the entire quiz flow; tight coupling prevents independent evolution
- **Fix:** Extract concerns; QuizNotifier should delegate to a single orchestrator
- **Effort:** 4h

### Finding: CategoriesScreen Provider Invalidation Storm
