## The "Closed-Loop" Logic

### Article-Quiz Relationship
The `QuizTable` (local) and `quiz_table` (remote) contain a `article_id` column that references `Articles.id`. This creates a direct link from each quiz question to its parent article.

```sql
quiz_table.article_id â†’ articles.id
```

### Mistake-Driven Learning
The `wrong_count` column drives the "Mistakes" queue:

1. **On incorrect answer**: `wrong_count` is incremented
2. **On correct answer**: If `quality >= 3`, `wrong_count` decrements by 1
3. **Mistakes queue**: Questions with `wrong_count > 0` are prioritized for review
4. **SM-2 Algorithm**: Controls scheduling via `easeFactor`, `srInterval`, `repetitions`, and `nextDueAt`

The system forms a closed loop where quiz performance directly influences future article study focus.

## Sync Strategy

### Incremental Sync
Uses `updatedAt` cursor-based incremental sync:

1. **Local cursor**: Each table has `updatedAt` column mirroring Supabase `updated_at`
2. **Remote comparison**: Sync fetches records where `updated_at > last_sync_cursor`
3. **Merge strategy**: `insertOnConflictUpdate` - upserts based on `remote_id` for quiz data, `id` for articles

### Drift Integration
```dart
// Example upsert pattern
await into(quizTable).insertOnConflictUpdate(QuizTableCompanion(...));
```

The sync runs automatically on app start when connectivity is available, and can be triggered manually via pull-to-refresh.

## Security Layers

### 1. R8 Obfuscation
- **ProGuard rules**: `android/app/proguard-rules.pro` for Drift R8 compatibility
- **enableProguard**: Enabled in `build.gradle.kts` release builds
- **Code shrinking**: Enabled to reduce APK size and obfuscate logic

### 2. Certificate Pinning (Dio)
- All Supabase/Dio HTTP requests go through pinned SSL certificates
- Prevents man-in-the-middle attacks on API calls
- See `lib/core/network/` for interceptor implementation

### 3. Runtime Signature Lock
- **MethodChannel**: `com.wardready.app/security`
- **SHA-256 hash verification**: Compares signing certificate against `expectedSignatureHash` in `SecurityService`
- **Sync disabled on failure**: If signature mismatch, `isSyncAllowed = false` and remote sync is blocked
- **Signature source**: Copied from "System Health" screen after release build installation

### 4. Row Level Security (Supabase)
- See `SUPABASE_SECURITY.sql` - execute before release
- Anon role: SELECT-only access to `articles`, `quiz_table`, `flashcard_table`
- INSERT/UPDATE/DELETE blocked for anon role
